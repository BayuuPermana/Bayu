# [t4_others] Refactor Remaining Components Implementation Plan

## Overview
This plan addresses the remaining "AI slop" in secondary components (Contact, Navigation, Projects, TechStack). We will replace generic marketing copy, remove redundant animations, and standardize styling to match the professional tone established in the Hero and About components.

## Scope Definition (CRITICAL)
### In Scope
- Refactor `components/Contact.tsx`.
- Refactor `components/Navigation.tsx`.
- Refactor `components/Projects.tsx`.
- Refactor `components/TechStack.tsx`.
### Out of Scope (DO NOT TOUCH)
- Global CSS (t5_global).
- Changing component logic/functionality.

## Current State Analysis
- `Contact.tsx`: Redundant large shadow on buttons (L18).
- `Navigation.tsx`: Gradient logo text (L41).
- `Projects.tsx`: Superlative-heavy descriptions and blur-heavy hover effects.
- `TechStack.tsx`: Brittle marquee animation and filler sub-header.

## Implementation Phases

### Phase 1: Contact & Navigation Refinement
- **Goal**: Clean up branding and styling.
- **Steps**:
  1. [x] `Contact.tsx`: Remove `shadow-green-200` and `shadow-lg` from the email button. Use a more subtle shadow or none.
  2. [x] `Navigation.tsx`: Replace gradient text logo with solid color or more professional typography.
- **Verification**: Visual check.

### Phase 2: Projects Copy & UI Cleanup
- **Goal**: Professionalize project descriptions and simplify interactions.
- **Steps**:
  1. [x] `Projects.tsx`: Rewrite descriptions to be technical and outcome-focused.
  2. [x] `Projects.tsx`: Remove the "Preview Available" blur overlay and simplify the hover effect.
- **Verification**: Visual check.

### Phase 3: TechStack Simplification
- **Goal**: Replace marquee with a clean grid and remove filler copy.
- **Steps**:
  1. [x] `TechStack.tsx`: Replace the marquee animation with a static or more subtle grid/list of icons.
  2. [x] `TechStack.tsx`: Remove the "Core technical proficiencies in motion" sub-header.
- **Verification**: Visual check.

### Phase 4: Final Audit
- **Goal**: Ensure consistency across all refactored components.
- **Steps**:
  1. [x] Check all touched files for redundant Tailwind classes.
- **Verification**: `npm run build`.
